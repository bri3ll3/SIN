# 🧪 Proyecto SIN – Ejemplos de Aplicaciones Web


## 🎯 Propósito del repositorio


Este repositorio tiene un propósito **educativo**, orientado al módulo de *Sistemas Informáticos* dentro del CFGS DAW. Su objetivo es mostrar, de forma práctica y progresiva, cómo se construyen aplicaciones web reales combinando diferentes tecnologías.


A lo largo del proyecto se trabajan conceptos fundamentales como:


- Desarrollo cliente (HTML, CSS, JavaScript)
- Programación en servidor (PHP)
- Persistencia de datos (MySQL)
- Interacción cliente-servidor
- Aplicaciones dinámicas e interactivas


---


## 📂 Proyectos incluidos


### 🟢 Ejemplos básicos


- **`index.html`**  
  👉 *Ejemplo 1: HTML (cliente)*  
  Página web estática básica.


- **`2.php`**  
  👉 *Ejemplo 2: PHP básico*  
  Introducción a PHP y generación de contenido dinámico.


- **`3.php`**  
  👉 *Ejemplo 3: Interacción formulario + servidor (PHP)*  
  Envío de datos desde formulario y procesamiento en servidor.


- **`4.php`**  
  👉 *Ejemplo 4: CRUD con base de datos*  
  Aplicación completa con operaciones Create, Read, Update y Delete utilizando MySQL.


---


### 🎮 Proyectos interactivos (final del bloque)


- **`game1.php`**  
  👉 *Game 1: 🎮 Mini Juego: Clic y gana puntos*  
  Primer ejemplo de aplicación interactiva con JavaScript y almacenamiento de puntuaciones.


- **`game2.php`**  
  👉 *Game 2: 🎮 Arcade Click Game v2*  
  Versión más avanzada con temporizador, animaciones y ranking en base de datos.


- **`game3.php`**  
  👉 *Game 3: 🎮 Tetris estilo "Cyberpunk"*  
  Proyecto final más complejo con mayor dinamismo y lógica en el cliente.


---


## 🧠 Enfoque didáctico


Este repositorio sigue una progresión clara:


1. Cliente (HTML)
2. Servidor (PHP)
3. Interacción (formularios)
4. Persistencia (base de datos)
5. Aplicaciones completas (CRUD)
6. Aplicaciones dinámicas (juegos)


El objetivo no es solo entender el código, sino **comprender cómo interactúan todas las partes de una aplicación web moderna**.


---


## ⚙️ Requisitos


Para ejecutar los ejemplos se recomienda un entorno como:


- Laragon (o similar)
- PHP
- MySQL
- Navegador web


---


## 🚀 Uso


1. Clonar el repositorio
2. Copiar los archivos en el directorio `www` de Laragon
3. Crear las bases de datos necesarias
4. Acceder desde el navegador (por ejemplo: `http://sin.test`)


---


## 📚 Nota final


Este proyecto está diseñado como material de aprendizaje, por lo que el código prioriza la claridad y la comprensión frente a la optimización o complejidad.
